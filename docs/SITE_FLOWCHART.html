Ingelogde gebruiker

Drie sub-types: gratis (geen aankopen), gratis (met losse aankopen), premium abonnee.

Gratis gebruiker (geen aankopen)

Homepage: "Welkom terug, [naam]" + badge "Gratis" + lege staat: "Je hebt gratis toegang tot samenvattingen" + CTA naar /analyses en /prijzen

Analyse detail: Samenvatting + Bedrijf + Bronnen zichtbaar. Rest: blur + "Bekijk prijzen" → /prijzen

Gratis + losse aankopen

Homepage: "Welkom terug" + kaarten van gekochte analyses (max 3)

Gekochte analyse: ALLES zichtbaar (10 secties)

Niet-gekochte analyse: Blur + "Bekijk prijzen"

Premium abonnee

Homepage: "Welkom terug" + badge "Premium" + alle analyse-kaarten (max 3, link naar rest)

Elke analyse: ALLES zichtbaar, geen paywall

BETAALFLOW
Stripe Checkout (extern)
Betaling
  • Gebruiker verlaat de site → Stripe hosted page
  • Losse analyse (€4,95): mode=payment
  • Premium maand (€9,95): mode=subscription
  • Premium jaar (€99,50): mode=subscription
Na betaling

Succes:

  • Losse analyse → /analyse/[ticker]?betaald=1
  • Abonnement → /account?welkom=1

Geannuleerd:

  • Losse analyse → /analyse/[ticker]
  • Abonnement → /prijzen

Achter de schermen: Stripe webhook → DB update (purchase of subscription)

Achter de schermen
Stripe Webhooks
  • checkout.session.completed → Purchase of Subscription aanmaken
  • customer.subscription.updated → Plan/status updaten
  • customer.subscription.deleted → Terug naar GRATIS
  • invoice.payment_failed → Status PAST_DUE
Email-notificaties (Resend) LIVE

Admin krijgt email bij:

  • Nieuwe registratie (naam + email)
  • Losse aankoop (ticker + email)
  • Nieuw abonnement (plan + email)
  • Opzegging (email)

Fire-and-forget, blokkeert nooit de user flow. Afzender: onboarding@resend.dev (eigen domein na verificatie).

NEXT_PUBLIC_UNLOCK

In .env.local staat NEXT_PUBLIC_UNLOCK=1

Dit schakelt de PaywallGate volledig uit — alle content is zichtbaar zonder betaling.

Check: staat dit ook op Vercel? Dan ziet iedereen alles gratis.
